Data at Rest

Refers to inactive data which is stored physically in any digital form (e.g. harddisk, USB drive, CDs and DVDs etc.).

Provides only minimal details on cryptography and security implementations -20 points

Good security is based on correct implementation of well established and tested security principals. It does not rely on the secrecy of technologies and designs used. By publishing an overview of its security architecture, a company offers its users transparency and allows the security industry to objectively review the safety of the service. Not publishing details forces users to make an ill-informed choice, potentially putting them at risk.

Website only contains a paragraph along the lines of "we use bank-grade security"

No zero-knowledge policy -15 points

This service stores user data in an encrypted form but has access to the master key which allows them to decrypt the data. This potentially exposes the user's data to rogue employees, government requests (including foreign governments if the data is hosted outside your country), and criminals in the event of a successful security breach.

Data is encrypted using 256 bit AES but they have access to master key to allow support to troubleshoot issues.

Secure password storage +15 points

This service uses a strong and well established method of securely storing passwords.

This service uses bcrypt

Technical specifications

Authentication & Session Management

Authentication is proving you are who you say you are and session management is the process of maintaining that trust once established. Most website and services require you to register your details and then log in to use the service, often using a username and password.

Only a limited choice for authenication and the password reset method is weak.

No two-factor support -5 points

Two-factor adds an additional authentication method to existing methods such as username and password. Passwords provide a 'something you know' whereas two-factor often provides a 'something you have'. Even if a password is compromised, the account cannot be access without the second authentication method. Hardware tokens, SMS messages and mobile phone applications are often used as the second factor. For additional security, the second-factor tokens often have a very short life span so cannot be recorded and used later.

No federated authentication -5 points
